For each of the 8 command files in `src/templates/claude/commands/codewiki/` (absorb.md, breakdown.md, ingest.md, lint.md, prd.md, process.md, query.md, tasks.md):

1. Read the original command file content.
2. Create a new directory `src/templates/skills/codewiki-<name>/` where `<name>` matches the filename stem (e.g., `ingest.md` becomes `codewiki-ingest/SKILL.md`).
3. Create `SKILL.md` in that directory with the following changes to the YAML frontmatter:

   - REPLACE the existing `description:` field with a richer description that helps the AI agent auto-match natural-language requests to this skill. The enriched description must:
     - Start with a clear action verb (Digest, Search, Check, Extract, Find, Create, Generate, Process)
     - Include enough context for the agent to distinguish this skill from the other 7
     - Stay under ~150 characters for scannability
   - PRESERVE all other frontmatter fields exactly: `allowed-tools`, `argument-hint`.
4. Preserve the full body content of each command file unchanged (all `<purpose>`, `<process>`, approval gates, backlinks references, etc.).

The 8 enriched descriptions should be:
- codewiki-ingest: "Digest a raw source document into wiki pages by extracting entities, decisions, lessons, and issues, then proposing updates for human approval"
- codewiki-query: "Search the wiki for relevant pages using backlink-indexed ranking and synthesize an answer with source citations"
- codewiki-lint: "Check the wiki for contradictions, orphans, stale content, anti-cramming and anti-thinning violations, and file drift against tracked hashes"
- codewiki-absorb: "Extract durable wiki knowledge from recent git changes using diff context and backlinks, proposing updates for human approval"
- codewiki-breakdown: "Find referenced-but-undocumented wiki entities and rank them by backlink importance, proposing new pages for human approval"
- codewiki-prd: "Create a product requirements document through structured clarifying questions, producing a PRD file in the tasks directory"
- codewiki-tasks: "Generate a task breakdown from a PRD with a mandatory Go gate before execution, producing a task file in the tasks directory"
- codewiki-process: "Process a task list one sub-task at a time, verifying each step, committing progress, and pausing for human approval between sub-tasks"

Do NOT delete the old `src/templates/claude/commands/` directory yet. It stays until Plan 02 updates the adapter to reference the new path. This avoids breaking the build mid-migration.
