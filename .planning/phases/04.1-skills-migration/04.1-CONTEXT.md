# Phase 04.1: Skills Migration - Context

**Gathered:** 2026-04-11
**Status:** Ready for planning

<domain>
## Phase Boundary

Phase 4 shipped Claude integration to `.claude/commands/codewiki/`, but the v2 canon requires eight standalone Skills, one per logical command. Phase 04.1 corrects that shipped surface and aligns installer output, tests, and planning docs to the verified skills model.

</domain>

<decisions>
## Implementation Decisions

### Skills canon
- **D-01:** CodeWiki installs eight standalone Skills, not slash commands and not one bundled wiki skill.
- **D-02:** Claude Code uses `.claude/skills/codewiki-<name>/SKILL.md`; Codex, Copilot, and OpenCode use `.agents/skills/codewiki-<name>/SKILL.md`; combined selections use both trees.
- **D-03:** Use the globally installed Claude skill `/create-skill` to create each of the eight CodeWiki skills. Planning should call this out explicitly instead of assuming the skill files will be hand-authored.

### Scope boundary
- **D-04:** Hook scripts and agent definitions remain shared assets; this phase changes the skill install surface and its code/doc/test fallout, not the hook-script feature set.
- **D-05:** `docs/skills/wiki.md` remains a format reference only; CodeWiki does not bundle subcommands into a single skill.

</decisions>
