# Phase 04.1.1: Skill Template Source - Context

**Gathered:** 2026-04-11
**Status:** Ready for execution planning
**Parent Phase:** 04.1 Skills Migration

<domain>
## Phase Boundary

This sub-phase creates the new canonical source templates for the eight CodeWiki skills under `src/templates/skills/codewiki-<name>/SKILL.md`.

This sub-phase does **not** touch adapter code, tests, packaging assertions, or docs. Those belong to 04.1.2 through 04.1.5.
</domain>

<decisions>
## Implementation Decisions

- **D-01.1:** Preserve the existing prompt intent from the slash-command source templates. Keep the current `# CodeWiki ...` heading, `purpose` / `process` XML sections, guardrails, approval gates, and tool allowances unless a skill-format requirement forces a small metadata change.
- **D-01.2:** Each migrated skill file must include `name`, `description`, and `argument-hint` frontmatter in addition to the existing `allowed-tools` metadata.
- **D-01.3:** Use `argument-hint: ""` when the current command does not define a required argument contract. Do not invent new invocation semantics in this sub-phase.
- **D-01.4:** Leave the legacy files in `src/templates/claude/commands/codewiki/` untouched during 04.1.1 so the current installer and tests keep working until 04.1.2 and 04.1.3 rewire consumers.
- **D-01.5:** `/create-skill` remains a quality reference only. These files are migrated by hand from the existing command templates.
</decisions>
