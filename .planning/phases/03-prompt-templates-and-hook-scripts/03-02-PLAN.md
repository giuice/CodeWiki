Create directory `src/templates/hooks/` and file `pre-wiki-context.sh`.

The script MUST (per D-08):
1. Start with `#!/bin/sh` shebang
2. Include `trap 'exit 0' EXIT` to guarantee exit 0
3. Include `set -e` after the trap
4. Read `wiki/index.md` -- if file does not exist, exit 0 silently
5. Output `## CodeWiki Context` header followed by wiki/index.md content
6. Grep wiki/index.md for terms related to the current context and output matching page summaries

POSIX compliance rules (HOOK-04):
- Use `[ ]` not `[[ ]]`
- Use `$(command)` not backticks
- Do NOT use `local` keyword (use unique variable names like `_cwiki_index`)
- Use `printf` not `echo -e`
- Quote all variable expansions

Script structure:
```sh
#!/bin/sh
# codewiki: pre-wiki-context hook
# Reads wiki/index.md and outputs relevant context to stdout
# Always exits 0 -- never blocks the agent
trap 'exit 0' EXIT
set -e

_cwiki_index="wiki/index.md"

if [ ! -f "$_cwiki_index" ]; then
    exit 0
fi

printf '## CodeWiki Context\n\n'
cat "$_cwiki_index"
printf '\n'

# If stdin provides context about current file, grep for related entries
# The hook receives tool input on stdin; extract file references
_cwiki_input=$(cat 2>/dev/null) || _cwiki_input=""
if [ -n "$_cwiki_input" ]; then
    # Extract potential entity names from input
    _cwiki_terms=$(printf '%s' "$_cwiki_input" | grep -oE '[A-Za-z][A-Za-z0-9_-]+' | sort -u | head -20) || _cwiki_terms=""
    for _cwiki_term in $_cwiki_terms; do
        if grep -qi "$_cwiki_term" "$_cwiki_index" 2>/dev/null; then
            # Find the wiki page path from index
            _cwiki_page=$(grep -i "$_cwiki_term" "$_cwiki_index" | grep -oE 'wiki/[^ )]+\.md' | head -1) || _cwiki_page=""
            if [ -n "$_cwiki_page" ] && [ -f "$_cwiki_page" ]; then
                printf '\n### Related: %s\n' "$_cwiki_term"
                head -20 "$_cwiki_page"
                printf '\n'
            fi
        fi
    done
fi
```

Create `src/templates/hooks/post-verify.sh`.

The script MUST (per D-09):
1. Start with `#!/bin/sh` shebang
2. Include `trap 'exit 0' EXIT` then `set -e`
3. Read JSON payload from stdin: `_cwiki_payload=$(cat 2>/dev/null) || _cwiki_payload=""`
4. Exit 0 if payload is empty
5. Exit 0 if wiki/entities/ directory does not exist
6. Parse payload for file paths using jq with grep fallback:
   - If `jq` available: `printf '%s' "$_cwiki_payload" | jq -r '.. | strings' 2>/dev/null`
   - Else: `printf '%s' "$_cwiki_payload" | grep -oE '"[^"]+\.[a-zA-Z0-9]+"' | tr -d '"'`
7. For each wiki entity file in wiki/entities/*.md, extract entity name (basename without .md)
8. Check if any extracted file path contains the entity name (case-insensitive grep)
9. If match found, output: `Wiki entity "ENTITY_NAME" may need updating.`

POSIX compliance: same rules as Task 1 -- no `[[`, no `local`, no backticks, quote all vars.

Script structure:
```sh
#!/bin/sh
# codewiki: post-verify hook
# Checks if modified files relate to wiki entities; exits 0 always
trap 'exit 0' EXIT
set -e

_cwiki_entities="wiki/entities"
_cwiki_payload=$(cat 2>/dev/null) || _cwiki_payload=""

[ -z "$_cwiki_payload" ] && exit 0
[ ! -d "$_cwiki_entities" ] && exit 0

# Extract file paths from JSON payload
if command -v jq >/dev/null 2>&1; then
    _cwiki_files=$(printf '%s' "$_cwiki_payload" | jq -r '.. | strings' 2>/dev/null) || _cwiki_files=""
else
    _cwiki_files=$(printf '%s' "$_cwiki_payload" | grep -oE '"[^"]+\.[a-zA-Z0-9]+"' | tr -d '"') || _cwiki_files=""
fi

[ -z "$_cwiki_files" ] && exit 0

for _cwiki_efile in "$_cwiki_entities"/*.md; do
    [ -f "$_cwiki_efile" ] || continue
    _cwiki_ename=$(basename "$_cwiki_efile" .md)
    if printf '%s' "$_cwiki_files" | grep -qi "$_cwiki_ename"; then
        printf 'Wiki entity "%s" may need updating.\n' "$_cwiki_ename"
    fi
done
```
