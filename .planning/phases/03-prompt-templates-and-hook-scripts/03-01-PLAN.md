Each file MUST have YAML frontmatter (delimited by triple-dash lines) containing: `description`, `allowed-tools`, and `argument-hint` fields. See the RESEARCH.md "Pattern 1: Claude Code Slash Command Format" section for the exact format.

Each file MUST use `<purpose>` and `<process>` XML tags per D-01.

**ingest.md** (per D-05):
- `description: Digest a raw source document into wiki pages`
- `argument-hint: <source-file-path>`
- `allowed-tools: [Read, Write, Edit, Glob, Grep, Bash]`
- Process steps: (1) Read source file from $ARGUMENTS, ask user if no arg; (2) Read wiki/index.md for existing entities; (3) Extract entities, decisions, lessons, issues, source summaries; (4) Cross-reference against existing wiki pages to avoid duplicates; (5) Present proposed new pages and index.md updates to user; (6) Wait for user approval before writing; (7) Create approved pages using templates from .codewiki/templates/, update wiki/index.md
- NO automatic git commits per D-03

**query.md** (per D-06):
- `description: Search wiki and synthesize an answer from wiki pages`
- `argument-hint: <question>`
- `allowed-tools: [Read, Glob, Grep, Bash]` (read-only query)
- Process steps: (1) Read wiki/index.md; (2) Search for relevant pages by keyword matching against $ARGUMENTS; (3) Read matched wiki pages; (4) Synthesize answer citing specific wiki entries with file paths; (5) If no relevant pages found, say so explicitly
- This is a local markdown RAG pattern -- read files, don't call APIs

**lint.md** (per D-07):
- `description: Check wiki for contradictions, orphans, stale content, and drift`
- `allowed-tools: [Read, Glob, Grep, Bash]` (no Write/Edit -- lint is read-only)
- No argument-hint needed
- Process steps: (1) Read wiki/index.md; (2) Glob wiki/**/*.md for all pages; (3) Detect contradictions between pages (conflicting statements about same entity); (4) Detect orphaned pages not listed in index.md; (5) Detect stale content referencing deleted/renamed code files; (6) Detect missing cross-references; (7) Detect template drift (pages not matching .codewiki/templates/ structure); (8) Output structured report with severity levels
  </action>
