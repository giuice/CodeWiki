  <action>
Rewrite `src/commands/init.ts` to implement the full install pipeline. Keep the `InitOptions` interface and `parseTools` helper. Replace the `initCommand` body with:

1. **Parse flags** (keep existing logic for `--force`, `--name`, `--tool`).

2. **Determine tools** per D-05/D-09/D-10:
   - If `--tool` provided: use those tools directly
   - Else: run `detectTools(root)` to find tools present
   - If detected tools is empty AND `process.stdin.isTTY`:
     - Use `createInterface` from `node:readline/promises` to prompt: `"No AI tools detected. Install for:\n  1) claude-code\n\nEnter number or tool name: "`
     - Map `"1"` or `"claude-code"` to `["claude-code"]`
     - Invalid input: throw error with `--tool` guidance
   - If detected tools is empty AND NOT TTY:
     - Throw: `"No AI tools detected. Use --tool to specify: codewiki init --tool claude-code"`

3. **Resolve templateDir**: `path.resolve(import.meta.dirname, "..", "templates")` — this resolves to `dist/templates/` at runtime since init.ts compiles to `dist/commands/init.js`.

4. **Scaffold wiki** (tool-independent): Call `scaffoldProject({ root, projectName, tools, force })`. Collect report entries.

5. **Run adapters**: Call `resolveAdapters(tools)`. For each adapter, call `adapter.install({ root, projectName, force, templateDir })`. Collect report entries per adapter. Log unsupported tools in report as skipped entries.

6. **Format and return report**: Use `formatSectionedReport(projectName, sections)` where sections are:
   - `{ title: "Wiki scaffold", entries: scaffoldEntries }`
   - One section per adapter: `{ title: "{tool} adapter", entries: adapterEntries }`
   - If unsupported tools: `{ title: "Unsupported (not yet implemented)", entries: unsupported.map(t => ({ action: "skipped", path: t, reason: "adapter not implemented" })) }`

The function signature remains: `export async function initCommand({ root, args }: InitOptions): Promise<string>`
