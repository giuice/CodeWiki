<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user project config → adapter merge logic | OpenCode installs modify `opencode.json` and `AGENTS.md` inside an existing user project |
| installed hook command → OpenCode runtime shell execution | The merged hook command string is executed by OpenCode after sessions complete |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-02-01 | Tampering | `opencode.json` merge in `src/lib/adapters/opencode.ts` | mitigate | Use `deepMerge` against parsed existing config so unrelated user keys survive and only `experimental.hooks.session_completed` is added or updated |
| T-06-02-02 | Denial of Service | duplicate OpenCode hook registrations on rerun | mitigate | Reuse `deduplicateHookEntries` and exact command-string matching so `session_completed` remains a singleton after repeated installs |
| T-06-02-03 | Elevation of Privilege | merged hook command string | mitigate | Use the constant `bash .codewiki/hooks/post-verify.sh` command only; do not interpolate user-controlled values into the hook command |
</threat_model>
