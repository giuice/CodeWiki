**2. Create `src/templates/hooks/session-end.sh`:**

Create a new POSIX shell script with the following content:

```sh
#!/bin/sh
# codewiki: session-end hook
# Outputs a summary of changes made during the session.
# Designed to be lightweight — just gather context, let the AI reason.
# Always exits 0 so it never blocks the agent.

trap 'exit 0' EXIT
set -e

# Output session change summary for the wiki-updater agent
_cwiki_diff_stat=""
_cwiki_changed_files=""

# Get diff stat since session start (last 1 commit as approximation)
_cwiki_diff_stat=$(git diff --stat HEAD~1 2>/dev/null) || _cwiki_diff_stat=""
_cwiki_changed_files=$(git diff --name-only HEAD~1 2>/dev/null) || _cwiki_changed_files=""

# If no committed changes, check staged/unstaged
if [ -z "$_cwiki_diff_stat" ]; then
    _cwiki_diff_stat=$(git diff --stat 2>/dev/null) || _cwiki_diff_stat=""
    _cwiki_changed_files=$(git diff --name-only 2>/dev/null) || _cwiki_changed_files=""
fi

# If still nothing, check staged
if [ -z "$_cwiki_diff_stat" ]; then
    _cwiki_diff_stat=$(git diff --cached --stat 2>/dev/null) || _cwiki_diff_stat=""
    _cwiki_changed_files=$(git diff --cached --name-only 2>/dev/null) || _cwiki_changed_files=""
fi

[ -z "$_cwiki_diff_stat" ] && exit 0

printf 'CODEWIKI_SESSION_SUMMARY\n'
printf 'Changed files:\n%s\n' "$_cwiki_changed_files"
printf 'Diff summary:\n%s\n' "$_cwiki_diff_stat"
printf 'END_CODEWIKI_SESSION_SUMMARY\n'
```

The script must:
- Use `#!/bin/sh` shebang (POSIX)
- Prefix all variables with `_cwiki_` (matching existing hook convention from pre-wiki-context.sh)
- Exit 0 under all conditions (trap + set -e pattern from existing hooks)
- Output structured text with `CODEWIKI_SESSION_SUMMARY` delimiters
- Handle: no git repo, no changes, uncommitted changes, committed changes

Replace the loop starting at line 29 (`for _cwiki_entity_file in ...`) through line 35 (end of loop) with:

```sh
_cwiki_matched=""
for _cwiki_entity_file in "$_cwiki_entities"/*.md; do
    [ -f "$_cwiki_entity_file" ] || continue
    _cwiki_entity_name=$(basename "$_cwiki_entity_file" .md)
    if printf '%s' "$_cwiki_files" | grep -Fqi "$_cwiki_entity_name"; then
        _cwiki_matched="${_cwiki_matched}${_cwiki_entity_name}\n"

done

[ -z "$_cwiki_matched" ] && exit 0

printf 'CODEWIKI_CHANGE_CONTEXT\n'
printf 'Affected wiki entities:\n'
printf '%b' "$_cwiki_matched"
printf 'Action: Run /codewiki-absorb to extract knowledge from these changes\n'
printf 'END_CODEWIKI_CHANGE_CONTEXT\n'
```

This gives the wiki-updater agent structured, parseable output with `CODEWIKI_CHANGE_CONTEXT` delimiters and a concrete action directive instead of a vague reminder.
